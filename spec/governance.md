# Governance 规范（治理）

> 跨层治理：隐私分级、新鲜度、证据、授权。v0.1 为**约定文档**，不做强制校验工具（工具化见 v0.2+）。

## 1. 敏感度四级
| 级别 | 含义 | 示例 | 默认策略 | 可收紧为 |
|---|---|---|---|---|
| L0 | 公开 | 公开社媒/个人网站 | 全放行 | 全放行 |
| L1 | 低敏 | 尺码/设备型号/日程偏好 | 全放行 | 全放行 |
| L2 | 中敏 | 常驻地/履历细节/通勤 | 全放行 | 仅本地模型 |
| L3 | 极敏 | 病史/资产/保单/亲友档案 | 全放行（用户可改） | 本地模型 + 显式授权 |

收紧只改一张访问矩阵表（`profile/console/agent-contract.md`），不动数据文件。

## 2. 新鲜度治理
- `verified`（最后核验日）+ `refresh`（周期）。
- facet 半衰期（借 OpenHuman）：身份 90d / 否决 60d / 风格 14d / 渠道 7d / 资产订阅 30d。
- 周期核验循环（v0.2）：扫超期文件，L1/L2 提醒、L3+agent 提议更新。

## 3. 证据分级（借 immortal）
- verbatim（原话）> artifact（成稿）> impression（旁人印象）。
- 矛盾不静默覆盖：旧值移历史 + 新值带时间戳 + 标来源。

## 4. 授权（六问，借 immortal，蒸馏他人时适用）
能否蒸馏 / 能否存储 / 能否商用 / 能否分享 / 保留期限 / 可否撤回。
蒸馏"自己"默认全允许；蒸馏他人必须显式授权。

## 5. 设计红线
1. 一切皆 markdown + git，可审计
2. 本地优先，云可选
3. 事实/知识/人格三维分层不混淆
4. 标准写进库，不绑工具
