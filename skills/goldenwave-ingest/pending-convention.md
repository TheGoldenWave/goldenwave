# _pending/ 草稿区约定

摄入路由器的"安全阀"。任何**低置信度或敏感**的候选页先落到目标库的 `inbox/_pending/`，等人确认后再移入正式目录。

## 何时进 _pending
- route 置信度 `< 0.7`（分类不确定）
- 敏感度 `≥ L2`（中/极敏事实，如财务、社交、法务）
- 目标层或域有多个候选，agent 无法裁定

## 草稿页格式
每个草稿页顶部必须带「路由决策卡」：
```
> 🧭 路由决策｜layer=L2/concept｜confidence=0.62｜进草稿原因：置信度<0.7
> 候选：concept vs insight ；建议 tags：技术/Agent
> 确认请移至 wiki/concepts/ 并删除本卡；改判请直接编辑后再移动。
```

## 人的三种处置
1. **采纳**：删决策卡 → 移到卡里指明的目标目录 → agent 下次会补索引/双链/log。
2. **改判**：直接编辑 frontmatter/正文/目标层，再移动。
3. **丢弃**：删除该草稿文件。

## 红线
- agent 不得自行把 `_pending/` 内容移入正式目录（那是人的决定）。
- `_pending/` 不进主知识图谱、不参与 lint 统计。
